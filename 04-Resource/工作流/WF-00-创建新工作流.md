# WF-00 创建新工作流

## 概述
新建工作流的标准流程，确保每个工作流都有完整的数据来源、输出模板和执行流程。

## 执行步骤

### 第一步：生成模板
```
判断是否需要外部数据源 or 输出模板
├── 需要 → 创建数据源文件: 04-Resource/数据源/<名称>.md
├── 需要 → 创建输出模板: 04-Resource/templates/<名称>-template.md
└── 不需要 → 跳过
```

### 第二步：创建工作流详情
创建文件 `04-Resource/工作流/WF-XX-工作流名称.md`，包含：

```markdown
---
title: "WF-XX 工作流名称"
created: YYYY-MM-DD
type: workflow-detail
tags: [workflow]
---

# WF-XX 工作流名称

## 概述
一句话说清这个工作流做什么

## 数据来源
- 数据源文档：`04-Resource/数据源/xxx.md`
- 或：无外部数据来源，内容来自本地/会话

## 输出模板
模板文件：`04-Resource/templates/xxx-template.md`

## 执行步骤
1. 
2. 
3. 

## 关联文件
- 数据源: `04-Resource/数据源/xxx.md`
- 模板: `04-Resource/templates/xxx-template.md`
- 输出: `01-Daily/xxx.md` 或 `03-Knowledge/xxx.md`
```

### 第三步：更新工作流总览
将新工作流添加到 `03-Knowledge/workflows.md`：
1. 工作流总览表（编号、名称、类型、归档、推送）
2. 关联路径表（模板、归档路径、推送范围）
3. 工作流详情表（详情文件、数据源、模板）
4. 操作日志

### 第四步：更新主页
在 `index.md` 的「快速链接」中确认工作流文档可达。

- [ ] 工作流详情文件已创建
- [ ] workflows.md 总览已更新
- [ ] workflows.md 关联路径已更新
