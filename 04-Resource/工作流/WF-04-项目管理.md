# WF-04 项目管理

## 目的

对进行中的项目进行统一管理，明确项目目标、进度、风险和产出，确保项目有序推进。

## 触发条件

- 新项目启动时
- 项目阶段里程碑完成时
- 每周项目回顾
- 项目状态变更时（延期、风险、完成等）

## 执行步骤

1. **项目初始化**：在 `03-Projects/` 下创建项目目录和 README
   - 明确项目目标、范围、时间线
   - 列出关键干系人和角色
   - 拆分里程碑和阶段性目标
2. **日常跟进**：
   - 更新任务状态（待办、进行中、已完成）
   - 记录关键决策和变更
   - 标记风险和阻塞项
3. **阶段回顾**：
   - 总结阶段成果和经验教训
   - 调整后续计划
   - 归档阶段性文档
4. **项目收尾**：
   - 编写项目总结报告
   - 归档全部项目文档至 `02-Archive/`
   - 提取可复用的知识沉淀至 `01-Knowledge/`

## 模板路径

`04-Resource/模板/项目README模板.md`

## 归档路径

- 进行中：`03-Projects/{项目名}/`
- 已完成：`02-Archive/项目/{项目名}/`

## 注意事项

- 每个项目必须有 README 作为入口文档
- 任务拆分应具体可执行，避免模糊描述
- 风险和阻塞项应第一时间记录并通知相关人
- 项目文档与知识库保持双向链接
- 完成的项目经验应及时沉淀为知识笔记
