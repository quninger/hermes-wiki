# WF-08 求职准备工作流

## 目的

系统化准备求职过程，包括简历优化、公司调研、面试准备、复盘总结，提高求职效率和成功率。

## 触发条件

- 开始新的求职周期时
- 收到面试邀请时
- 需要更新简历时
- 用户主动发起求职准备时

## 执行步骤

### 步骤 1：简历优化

1. **内容审查**
   - 检查简历格式和排版
   - 确保关键信息突出（技术栈、项目经验、成果）
   - 使用 STAR 法则优化项目描述

2. **关键词优化**
   - 分析目标职位 JD（职位描述）
   - 提取关键词并融入简历
   - 确保 ATS（简历筛选系统）友好

3. **量化成果**
   - 将模糊描述转化为具体数据
   - 示例：「优化性能」→「优化查询性能，响应时间降低 60%」

4. **工具支持**
   - 使用 `chinese-resume-pdf` 技能生成专业 PDF
   - 使用 `chinese-resume-optimization` 技能优化内容

### 步骤 2：目标公司调研

1. **公司背景**
   - 公司规模、融资阶段、业务领域
   - 技术栈和工程文化
   - 最近动态和新闻

2. **职位分析**
   - 详细解读 JD 要求
   - 识别核心技能和加分项
   - 评估匹配度和差距

3. **整理调研笔记**
   ```markdown
   # {公司名称} 调研

   ## 基本信息
   - 规模：
   - 融资：
   - 业务：

   ## 技术栈
   - 语言：
   - 框架：
   - 工具：

   ## 职位要求
   - 核心要求：
   - 加分项：
   - 匹配度：{X}%

   ## 准备重点
   {需要重点准备的内容}
   ```

### 步骤 3：面试准备

1. **技术面试**
   - 复习核心知识点
   - 准备项目经验介绍（STAR 格式）
   - 练习算法和数据结构题目
   - 准备系统设计讨论

2. **行为面试**
   - 准备常见问题：
     - 自我介绍
     - 优缺点
     - 团队协作经历
     - 遇到的挑战及解决
     - 职业规划
   - 使用 STAR 法则组织答案

3. **模拟面试**
   - 使用 Hermes 进行模拟面试
   - 记录回答并优化
   - 练习表达清晰度和逻辑性

### 步骤 4：面试记录

1. **面试前**
   - 确认面试时间、地点/链接
   - 准备问题清单（向面试官提问）
   - 检查设备和网络（线上面试）

2. **面试中**
   - 记录关键问题（如允许）
   - 注意面试官反馈

3. **面试后**
   - 及时记录面试内容：
   ```markdown
   # {公司} - {职位} 面试记录

   ## 基本信息
   - 日期：
   - 轮次：
   - 面试官：

   ## 问题记录
   1. {问题1}
      - 我的回答要点：
      - 反思：

   ## 感受
   - 整体表现：
   - 改进点：

   ## 后续
   - 结果：
   - 经验教训：
   ```

### 步骤 5：复盘优化

1. **每轮面试后**
   - 分析表现优劣
   - 识别知识盲区
   - 调整准备策略

2. **求职周期结束后**
   - 总结整体经验
   - 更新面试题库
   - 优化简历和求职策略

### 步骤 6：归档管理

1. 保存所有求职相关文档到 `03-Archive/求职准备/`
2. 文件命名：`{YYYY-MM-DD}-{公司}-{职位}.md`
3. 分类管理：
   - `简历/`：简历及优化记录
   - `调研/`：公司调研笔记
   - `面试/`：面试记录
   - `复盘/`：阶段性总结

## 注意事项

- **隐私保护**：求职信息高度敏感，注意加密或限制访问
- **真实性**：简历内容必须真实，面试回答诚实
- **持续更新**：根据面试反馈持续优化简历和准备策略
- **心态管理**：求职是双向选择，保持积极心态
- **时间管理**：合理安排准备时间，避免过度焦虑
- **记录习惯**：详细记录每次面试，便于复盘和改进
- **工具利用**：善用 AI 工具辅助准备，但保持个人特色
