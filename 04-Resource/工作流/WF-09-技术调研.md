# WF-09 技术调研

## 🎯 工作流目的

系统化地进行技术方案调研，收集、整理和对比分析不同技术选型，为项目决策提供可靠的技术依据。确保调研过程可追溯、结论有据可依。

## ⚡ 触发条件

- 需要引入新的技术栈、框架或工具时
- 现有方案遇到瓶颈，需要寻找替代方案时
- 项目规划阶段需要技术可行性评估时
- 用户提出"调研XX技术"、"对比XX方案"等需求时

## 📋 执行步骤

### 第一步：明确调研范围
1. 与用户确认调研目标和关键问题
2. 定义评估维度（性能、成本、生态、学习曲线等）
3. 确定调研的深度和时间约束
4. 创建调研任务文档，记录目标和范围

### 第二步：信息收集
1. 搜索官方文档、技术博客、学术论文
2. 收集 GitHub Star 数、Issue 活跃度、社区规模
3. 查找相关 Benchmark 和性能测试数据
4. 记录各方案的核心特性和限制条件
5. 寻找实际生产环境案例和踩坑经验

### 第三步：方案对比分析
1. 建立多维度对比矩阵（表格形式）
2. 对每个方案进行优劣势分析
3. 评估与现有技术栈的兼容性
4. 分析迁移成本和风险
5. 给出初步推荐意见和理由

### 第四步：输出调研报告
1. 整理调研报告，包含：
   - 调研背景与目标
   - 候选方案概述
   - 多维度对比表
   - 各方案详细分析
   - 推荐方案及理由
   - 风险提示与后续建议
2. 将报告存档到指定路径
3. 向用户汇报调研结果，解答疑问

## 📝 备注

- 调研过程中注意信息的时效性，优先参考最近一年的内容
- 对于商业方案，需额外关注许可证和费用模型
- 如有 Demo 或 POC 需求，应单独创建 WF-11 编码任务
- 调研结论应标注置信度，区分"有明确结论"和"建议进一步验证"
- 涉及安全、合规的技术选型，务必标注相关风险
