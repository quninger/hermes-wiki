# WF-05 会话归档工作流

## 目的

将 Hermes Agent 中的重要会话记录进行结构化归档，保存关键决策、解决方案和工作上下文，便于日后检索和知识复用。

## 触发条件

- 完成一个复杂任务（5+ 轮对话）后
- 发现重要技术方案或解决方案时
- 用户主动要求归档当前会话时
- 每周定期检查并归档有价值会话时

## 执行步骤

### 步骤 1：筛选会话

1. 使用 `session_search()` 浏览近期会话
2. 根据以下标准判断是否值得归档：
   - 解决了非显而易见的问题
   - 产生了可复用的方案或代码
   - 包含重要决策及其理由
   - 涉及复杂调试过程

### 步骤 2：提取关键信息

1. 使用 `session_search(session_id=...)` 读取完整会话
2. 提取以下要素：
   - **问题描述**：会话开始时的问题/目标
   - **关键决策**：过程中做出的重要选择
   - **解决方案**：最终采用的方案及步骤
   - **代码片段**：有价值的代码（如有）
   - **踩坑记录**：遇到的问题及解决方法
   - **结论**：最终结果和经验教训

### 步骤 3：结构化撰写

使用模板格式撰写归档文档：

```markdown
# {会话主题}

## 背景
{问题背景描述}

## 过程
{关键步骤和决策记录}

## 结果
{最终成果}

## 经验
{可复用的经验和教训}
```

### 步骤 4：保存归档

1. 将文档保存到 `03-Archive/会话归档/` 目录
2. 文件名格式：`{YYYY-MM-DD}-{简短主题}.md`
3. 添加 YAML frontmatter 标记元数据

### 步骤 5：索引更新

1. 检查是否需要更新知识图谱索引
2. 如有必要，使用 `skill_manage` 保存为可复用技能


## 注意事项

- **隐私保护**：归档时移除敏感信息（密码、密钥、个人信息）
- **简洁优先**：只保留有价值的内容，去除冗余对话
- **上下文完整**：确保归档内容在脱离原始会话后仍可理解
- **定期清理**：每季度检查归档，合并重复或过时内容
- **命名规范**：文件名使用中文，简洁明了，便于检索
