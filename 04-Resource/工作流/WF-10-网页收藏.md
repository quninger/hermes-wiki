# WF-10 网页收藏

## 🎯 工作流目的

将有价值的网页内容结构化地保存到知识库中，便于日后检索和引用。通过统一的标签体系和分类标准，建立个人的网页收藏知识图谱。

## ⚡ 触发条件

- 用户分享了一个或多个网页链接需要保存时
- 浏览过程中发现有参考价值的技术文章、教程时
- 需要整理一批收藏链接到知识库时
- 用户提出"收藏这个网页"、"保存这篇文章"等需求时

## 📋 执行步骤

### 第一步：内容提取与解析
1. 获取网页 URL 并访问页面
2. 提取页面核心内容：
   - 标题、作者、发布日期
   - 正文摘要（前 300 字关键内容）
   - 关键图片或图表描述
   - 页面类型判断（文章/教程/工具/文档/讨论）
3. 检测页面是否为动态加载内容，必要时使用浏览器工具

### 第二步：分类与标签
1. 根据内容主题分配主分类：
   - 技术开发（前端/后端/DevOps/数据库等）
   - 产品设计（UI/UX/产品思维等）
   - 行业资讯（AI/区块链/云计算等）
   - 学习资源（教程/课程/书籍等）
   - 工具效率（开发工具/效率软件等）
2. 添加细粒度标签（3-5 个关键词标签）
3. 标注重要程度（⭐~⭐⭐⭐⭐⭐）

### 第三步：格式化存储
1. 按照收藏模板生成 Markdown 文件
2. 包含以下字段：
   - 原始链接
   - 收藏日期
   - 内容摘要
   - 分类与标签
   - 个人备注
3. 保存到对应分类目录

### 第四步：索引更新
1. 更新收藏索引文件（按日期和分类）
2. 如有 Obsidian 笔记系统，同步创建双向链接
3. 记录收藏统计信息

```

```


索引文件：
```

```

## 📝 备注

- 对于需要登录才能查看的内容，标注访问限制
- 付费内容需标注价格和是否已购买
- 定期检查收藏链接的有效性，标记失效链接
- 如内容涉及版权敏感信息，遵守隐私设置
- 批量收藏时（>10 条），建议创建专用收集会话
- 对于长文章，建议额外提取"核心观点"摘要
