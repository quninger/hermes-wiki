# WF-12 模型编排执行细则（v4.0）

## 🎯 设计目标

基于 Hermes 原生 delegate_task，构建串行/并行/混合DAG工作流，实现任务难度动态分级、模型成本智能匹配。

**编排底座**：delegate_task
**MoA 定位**：仅用于局部多模型合议，不做通用编排

---

## 🏗 核心编排模式

### 模式1：串行流水线（强依赖）

适用：多步骤递进任务，后续步骤强依赖前置输出

```python
delegate_task(
  tasks=[
    {
      "id": "step1",
      "goal": "原始数据采集、素材整理",
      "model": "free-light-model"
    },
    {
      "id": "step2",
      "goal": "数据清洗、格式标准化",
      "depends_on": ["step1"],
      "model": "free-light-model"
    },
    {
      "id": "step3",
      "goal": "深度逻辑分析、结论生成",
      "depends_on": ["step2"],
      "model": "paid-high-model"
    }
  ]
)
```

### 模式2：并行多分支（无依赖）

适用：多维度独立子任务、批量信息搜集

```python
delegate_task(
  max_concurrent_children=3,
  tasks=[
    {"id": "p1", "goal": "基础信息搜集", "model": "free-light-model"},
    {"id": "p2", "goal": "格式校验", "model": "free-light-model"},
    {"id": "p3", "goal": "深度分析", "model": "paid-high-model"}
  ]
)
```

### 模式3：混合DAG（通用推荐）

拓扑：**串行预处理 → 并行分析 → 串行汇总**

```python
delegate_task(
  tasks=[
    # 前置串行：免费模型预处理
    {"id": "prep", "goal": "素材获取、数据预处理", "model": "free-light-model"},
    # 中间并行：按需区分模型
    {"id": "a1", "goal": "维度A分析", "depends_on": ["prep"], "model": "free-light-model"},
    {"id": "a2", "goal": "维度B分析", "depends_on": ["prep"], "model": "free-light-model"},
    {"id": "a3", "goal": "深度推理", "depends_on": ["prep"], "model": "paid-high-model"},
    # 汇聚：付费模型终审
    {"id": "final", "goal": "结果整合、逻辑校验", "depends_on": ["a1","a2","a3"], "model": "paid-high-model"}
  ]
)
```

## 📊 任务分级 & 模型路由

### 免费轻量模型（1~3分）

| 任务类型 | 具体场景 | 模型 |
|:---------|:---------|:-----|
| 信息检索 | 文档搜索、网页提取、资料汇总 | glm-4-flash |
| 文本处理 | 格式转换、压缩、排版 | glm-4-flash |
| 标准操作 | 模板填充、数据搬运、基础校验 | glm-4-flash |
| 方案规划 | 简单日程、步骤规划 | glm-4-flash |

### 付费高精度模型（4~5分）

| 任务类型 | 具体场景 | 模型 |
|:---------|:---------|:-----|
| 深度推理 | 方案设计、故障排查、代码优化 | deepseek-v4-flash |
| 多源融合 | 交叉验证、决策结论生成 | deepseek-v4-flash |
| 长文理解 | 深度阅读、复杂聚合 | mimo-v2.5-pro |
| 终审校验 | 结果整合、质量把关 | mimo-v2.5-pro |

### 路由执行策略

```
1. 所有请求入口 → 免费模型难度打分（1~5分）
2. 1~3分（简单）→ 全链路免费模型
3. 4~5分（复杂）→ 预处理用免费，核心/汇总用付费
4. 终审节点 → 高精度模型兜底
```

## 🔀 Delegate 与 MoA 职责边界

| 组件 | 职责 | 场景 |
|:----:|:-----|:-----|
| **delegate_task** | 通用编排底座 | 串行/并行/DAG、差异化任务、分级调度 |
| **MoA** | 局部合议增强 | 同一问题→多模型→聚合校验（降低幻觉） |

**推荐组合**：Delegate 完成 DAG 编排 → 关键节点嵌套 MoA 合议 → 后续继续串行

**禁止**：用 MoA 实现分步流水线、差异化子任务处理
