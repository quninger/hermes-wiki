# WF-12 模型编排执行细则（v6.0）

## 🎯 核心目标

在输出质量不变的前提下减少付费——充分利用免费模型，付费模型只做关键收口。

---

## 📋 免费模型能力矩阵

### Tier 1：轻量主力（GLM-4-flash）

| 能力 | 评分 | 说明 |
|:-----|:----:|:-----|
| 文本生成 | ⭐⭐⭐ | 日常问答、总结、翻译 |
| 信息检索 | ⭐⭐⭐ | 网页搜索、文档提取 |
| 格式转换 | ⭐⭐⭐ | JSON/Markdown互转 |
| 轻量编码 | ⭐⭐ | 简单脚本、代码片段 |

| **128k上下文** | ✅ | 足够处理长文档 |

**主力用途**：子代理默认模型、信息检索、格式化、轻量编码

## 🏗 编排方案（基于能力分级）

### 方案1：直接执行（分数1-5）

主代理直接做，不委派。

```
简单任务 → 主代理(deepseek/mimo)直接执行
```

### 方案2：并行批量（分数6-7/10）

所有子代理统一使用 **glm-4-flash（免费）**。

```
delegate_task(tasks=[
    {"goal": "子任务A", "toolsets": ["web"]},
    {"goal": "子任务B", "toolsets": ["terminal","file"]},
])
# 所有子代理 = glm-4-flash（免费）
```

### 方案3：串行分步（分数7-9）

收集用免费模型，实现用付费模型。

```
Step1: delegate(glm-4-flash) → 收集/探索（免费）
Step2: 主代理(deepseek) → 实现/修复（付费）
```

### 方案4：MoA 多视角（仅分析/评审）

```bash
# GLM-5.1/5.2 做参考（免费），Mimo聚合（付费）
hermes chat --provider moa -m standard -q "任务" -Q
```

### 方案5：混合流水线（机械+推理）

```
execute_code（机械收集，零LLM）
    → delegate(glm-4-flash)（初筛分类，免费）
        → 主代理(deepseek)（综合，付费）
```

## ⚙️ delegate_task 规范

### 模型配置（config.yaml 已生效）
```yaml
delegation:
  model: glm-4-flash      # 所有子代理统一免费
  provider: zai
auxiliary:
  vision: { model: glm-4-flash }    # 全部免费
  compression: { model: glm-4-flash }
  # ... 全部 auxiliary = glm-4-flash
```

### context 必填项
- 项目绝对路径
- 技术栈与环境
- 相关文件路径
- 验收标准

### 禁止项
- 禁止 leaf 子代理再 delegate
- 禁止 1-2 次工具调用开 delegate
- 禁止 子代理长输出全文返回

## 💰 成本对比

| 场景 | 旧方案 | 新方案 | 省多少 |
|:-----|:------:|:------:|:------:|
| 并行调研3方案 | 3×mimo | 3×**glm** | ~100% |
| 串行搜索+分析 | 2×mimo | 1×**glm** + 1×mimo | ~50% |
| 多文件审查 | N×mimo | N×**glm** | ~100% |
| 所有auxiliary | auto→mimo | **glm** | ~100% |
