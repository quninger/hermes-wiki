# WF-12 模型编排执行细则（v5.1）

## 🎯 核心目标

**在输出质量不变的前提下减少付费**——简单任务用免费模型，困难任务用付费模型。

**一句话原则**：免费模型（GLM）负责探索和副作用；付费模型（DeepSeek/Mimo）负责编排、实现与最终结论。

---

## 📋 执行优先级（强制，不可跳过）

收到用户任务后，**必须**按以下顺序决策：

1. 评估任务难度（1-10分）
2. 选择编排方案（方案1-5）
3. 选择模型
4. 执行任务
5. 付费模型收口质量
6. 恢复会话默认模型

---

## 🧠 难度评估（1-10分）

| 维度 | 低(1-3) | 中(4-7) | 高(8-10) |
|:-----|:--------|:--------|:---------|
| 工具调用次数 | ≤2 | 3-8 | >8或需并行 |
| 涉及文件数 | 0-1 | 2-5 | >5或跨模块 |
| 推理深度 | 事实/解释 | 多步探索 | 架构/安全/复杂bug |
| 可脚本化 | 是 | 部分 | 否 |
| 输出风险 | 低 | 中 | 高（生产代码） |

**综合分 = 各维度上限的加权直觉分；边界取更高分。**

## 📊 分数 → 路由（强制）

| 分数 | 编排方案 | 主模型 | 子任务模型 | MoA |
|:----:|:--------:|:------:|:----------:|:---:|
| 1-3 | 方案1/5 | 当前模型 | 不delegate | 否 |
| 4-5 | 方案1/5 | 当前模型 | 优先execute_code | 否 |
| 6-7 | 方案2/3 | 当前模型 | **必须glm-4-flash** | 否 |
| 8-9 | 方案3/4 | 当前模型 | 收集glm，实现付费 | 仅多视角时用 |
| 10 | 方案2+3+外部 | 当前模型 | 混合；必要Cursor CLI | deep preset |

---

## 🏗 五种编排方案

### 方案1：直接执行（分数1-5，≤5次工具调用）

- 主代理直接调用 terminal/file/web
- **禁止** delegate_task
- **禁止** MoA

### 方案2：并行批量（多个独立子任务，分数6-7/10）

```python
delegate_task(
  tasks=[
    {"goal": "子任务A", "context": "...", "toolsets": ["web"]},
    {"goal": "子任务B", "context": "...", "toolsets": ["terminal","file"]},
  ]
)
# 所有子代理自动使用 delegation.model（glm-4-flash，免费）
```

- **禁止** 批量任务全部用 DeepSeek（除非用户明确要求）
- 父代理只读各子代理**摘要**，禁止全文贴入

### 方案3：串行分步（步骤有依赖，分数7-9）

```
Step1: delegate_task(glm) → 收集/探索
Step2: 主代理读 Step1 摘要 → 决策后执行（付费模型）
```

- 收集/探索步骤 → glm（免费）
- 实现/安全/架构步骤 → 主代理（付费）亲自做
- 每步完成再开下一步

### 方案4：MoA 多视角聚合（仅分析/评审场景）

```bash
hermes chat --provider moa -m lightweight -q "任务" -Q  # GLM→DeepSeek
hermes chat --provider moa -m standard -q "任务" -Q   # GLM+DeepSeek→Mimo
hermes chat --provider moa -m deep -q "任务" -Q       # DeepSeek+Mimo→Mimo
```

- **禁止** 用于流水线、批量改文件
- 完成后必须恢复会话模型

### 方案5：混合流水线（先机械收集后推理）

```
execute_code（机械收集，零LLM）
    → delegate_task(glm)（可选：初筛分类）
        → 主代理（付费）综合/写最终输出
```

- 能写Python脚本的，**必须** execute_code
- 脚本只做机械活，判断留给付费模型

## ⚙️ delegate_task 强制规范

### 模型优先级（官方确认）
```
delegation.model (config.yaml)  →  父模型继承
```
**所有 delegate_task 子代理统一使用 delegation.model（glm-4-flash，免费）**

### context 必填项
每个 delegate_task 的 context **必须**包含：
- 项目绝对路径
- 技术栈与环境
- 相关文件路径列表
- 错误信息或需求原文
- 验收标准

### 禁止项
- 禁止 leaf 子代理再 delegate
- 禁止 1-2 次工具调用开 delegate
- 禁止 子代理长输出全文返回用户（只返回摘要+结论）

## 💰 成本 ↔ 质量平衡

### 免费模型允许做的事
- 网页/文档调研与整理
- 目录扫描、grep、只读分析
- 格式转换、changelog 草稿
- 子代理探索性工具调用

### 付费模型必须做的事
- 最终关键结论（架构、安全、生产代码）
- 复杂 bug 根因与修复
- MoA aggregator 层输出

### 质量门禁（输出前自检）
- [ ] 关键结论是否经过付费模型审核？
- [ ] 免费模型产出的代码是否已跑测试？
- [ ] 是否误把 MoA 用于流水线任务？

## 📋 典型任务速查

| 用户意图 | 分数 | 方案 | 模型 |
|----------|:----:|:----:|:-----|
| 解释概念/短问答 | 1-2 | 1 | 当前模型 |
| 读1个文件说明 | 2-3 | 1 | 当前模型 |
| 批量重命名/JSON解析 | 3-4 | 5 | execute_code |
| 调研3个技术方案 | 6-7 | 2 | 3×glm并行→综合 |
| 修单个测试失败 | 5-6 | 1/3 | DeepSeek |
| 多文件refactor | 8-9 | 3 | glm探索→DeepSeek实现 |
| 架构/security评审 | 8-9 | 4 | MoA standard/deep |
