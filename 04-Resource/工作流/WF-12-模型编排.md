# WF-12 模型编排

## 🎯 工作流目的

智能调度和编排多个 AI 模型及 Agent，根据任务类型、复杂度和资源约束，选择最优的模型组合和执行策略，实现高效、经济的 AI 协作工作流。

## ⚡ 触发条件

- 任务需要多个 AI Agent 协作完成时
- 需要根据任务特性选择最优模型时
- 批量处理任务需要并行调度时
- 用户提出"用XX模型处理"、"并行执行XX"等需求时
- 任务链需要前后依赖、多步骤编排时

## 📋 执行步骤

### 第一步：任务分析与分解
1. 分析任务的性质和要求：
   - 语言能力需求（中文/英文/多语言）
   - 推理深度（简单判断/复杂推理）
   - 创意需求（精确执行/创意生成）
   - 上下文长度需求
   - 多模态需求（文本/图片/代码/音频）
2. 将复杂任务分解为子任务
3. 识别子任务之间的依赖关系
4. 确定哪些子任务可以并行执行

### 第二步：模型选择与路由
1. 根据子任务特性匹配最优模型：
   - 代码任务 → Claude Code / Codex
   - 中文写作 → 擅长中文的模型
   - 数学推理 → 推理能力强的模型
   - 创意内容 → 创意能力突出的模型
   - 低成本任务 → 经济型模型
2. 考虑模型的 Token 限制和成本
3. 确定回退策略（主模型失败时的备选）
4. 记录模型选择决策和理由

### 第三步：执行编排
1. 构建执行计划（DAG 图）
2. 按依赖顺序调度子任务：
   - 无依赖任务 → 并行执行
   - 有依赖任务 → 等待前置完成后执行
3. 监控各子任务执行状态
4. 处理异常和超时：
   - 重试机制（指数退避）
   - 模型降级切换
   - 人工介入标记
5. 收集和整合中间结果

### 第四步：结果聚合与输出
1. 整合各子任务的输出结果
2. 进行一致性检查和质量验证
3. 格式化最终输出
4. 记录执行日志：
   - 各模型调用次数和 Token 消耗
   - 执行时间和延迟
   - 错误率和重试次数
5. 向用户交付结果

## 📝 备注

- 委托子 Agent 时使用 `task` 工具，明确指定子任务目标和约束
- 并行 Agent 数量建议不超过 5 个，避免资源争抢
- 模型切换时注意上下文连续性，必要时传递关键上下文
- 涉及敏感数据的任务，注意模型的数据隐私政策
- 定期评估模型编排效果，优化模型选择策略
- 参考 autonomous-ai-agents 技能组了解 Agent 委托最佳实践
- 模型编排的成本应可控，批量任务建议设置 Token 预算上限
- 对于实时性要求高的任务，优先选择响应快的模型
