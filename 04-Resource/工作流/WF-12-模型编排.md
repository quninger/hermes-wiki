# WF-12 模型编排执行细则（v5.0）

## 🎯 设计原则

基于 Hermes 官方文档实际能力设计，不假设不存在的功能。

**官方能力依据**：
- 官方文档：https://hermes-agent.nousresearch.com/docs/user-guide/features/delegation
- 中文指南：https://hermes-agent.nousresearch.com/docs/zh-Hans/guides/delegation-patterns

---

## 📋 官方能力清单

| 工具 | 能力 | 限制 |
|:----:|:-----|:-----|
| `delegate_task` | 并行批量执行（最多3路） | 所有子代理继承父模型，不能指定不同模型 |
| `execute_code` | 顺序执行机械步骤 | 无推理能力，只能调用工具 |
| `hermes chat --provider moa` | MoA 多模型聚合 | 需通过 terminal 调用，不同 preset 用不同模型 |
| `cronjob` | 定时任务，支持 model 参数 | 适合周期任务 |
| 主代理 | 编排决策、串行调度 | 就是当前会话模型 |

**不存在的能力**（官方文档确认）：
- ❌ `depends_on` 串行依赖
- ❌ 每任务独立 `model` 参数
- ❌ DAG 有向无环图调度

---

## 🏗 编排方案

### 方案1：主代理直接执行（简单任务）

适用：任务简单，当前模型（deepseek）可直接完成

```
用户任务 → 主代理(deepseek) 直接执行 → 输出
```

示例：文件读写、简单问答、格式转换

### 方案2：并行批量（delegate_task）

适用：多个独立子任务，无先后依赖

```
用户任务
    │
    ▼
delegate_task(tasks=[
    {goal: "子任务A", ...},
    {goal: "子任务B", ...},
    {goal: "子任务C", ...}
])
    │
    ▼
主代理汇总结果
```

示例：多文件代码审查、多维度并行调研

### 方案3：串行分步（主代理多次调度）

适用：步骤间有依赖，需要前一步结果

```
用户任务
    │
    ▼
主代理: delegate_task(tasks=[步骤1]) → 等待结果
    │
    ▼
主代理: 根据步骤1结果，delegate_task(tasks=[步骤2, 步骤3]) → 等待结果
    │
    ▼
主代理: 聚合所有结果 → 输出
```

示例：先搜索→再分析→最后归档

### 方案4：MoA 多模型聚合（通过 terminal）

适用：需要不同模型从不同角度分析同一问题

```
用户任务
    │
    ▼
主代理: terminal("hermes chat --provider moa -m deep -q '...' -Q")
    │
    ▼
MoA 内部: GLM+DeepSeek 参考 → Mimo 聚合
    │
    ▼
主代理: 收集 MoA 输出 → 归档
```

### 方案5：混合流水线（execute_code + delegate_task）

适用：先机械收集，再推理分析

```
用户任务
    │
    ▼
execute_code: 机械数据收集（搜索、提取、格式化）
    │
    ▼
delegate_task: 推理分析（深度分析、结论生成）
    │
    ▼
主代理: 聚合归档
```

示例：先搜索10个网页 → 再深度分析内容 → 最后归档

## 📊 任务-编排路由表

| 任务类型 | 难度 | 编排方案 | 执行工具 | 模型 |
|:---------|:----:|:---------|:---------|:-----|
| 文件读写/格式转换 | 轻量 | 方案1：直接执行 | 主代理 | deepseek |
| 简单问答/日常对话 | 轻量 | 方案1：直接执行 | 主代理 | deepseek |
| 批量文件处理 | 中等 | 方案2：并行批量 | delegate_task | deepseek(继承) |
| 多维度调研 | 中等 | 方案2：并行批量 | delegate_task | deepseek(继承) |
| 先搜索后分析 | 中等 | 方案5：混合流水线 | execute_code + delegate_task | deepseek |
| 串行归档流程 | 中等 | 方案3：串行分步 | 主代理多次调度 | deepseek |
| 需要多视角分析 | 中等 | 方案4：MoA聚合 | terminal(moa) | GLM+DeepSeek→Mimo |
| 复杂代码重构 | 困难 | 方案2：并行批量 | delegate_task | deepseek(继承) |
| 极困难任务 | 困难 | 方案1+外部工具 | terminal(Cursor CLI) | 外部 |

## 🔀 MoA 定位

**MoA 仅用于**：同一问题 → 多模型并行分析 → 聚合输出

**MoA 不用于**：任务拆解、差异化子任务、串行流水线

**调用方式**：
```bash
# 轻量（GLM参考 → DeepSeek聚合）
hermes chat --provider moa -m lightweight -q "任务" -Q

# 标准（GLM+DeepSeek参考 → Mimo聚合）
hermes chat --provider moa -m standard -q "任务" -Q

# 深度（DeepSeek+Mimo参考 → Mimo聚合）
hermes chat --provider moa -m deep -q "任务" -Q
```
