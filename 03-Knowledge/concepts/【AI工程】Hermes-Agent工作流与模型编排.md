## 🔄 二、标准化工作流（四步闭环法）

每次收到任务，按以下四步执行：

### 步骤 1: 匹配模板与路径
```
收到任务 → 查阅工作流表 → 匹配工作流 → 获取：
  ├── 资源来源文件（素材、数据）
  ├── 执行模板文件（输出格式）
  └── 结果归档路径（存储位置）
```

### 步骤 2: 按流程执行
- 严格遵循既定步骤
- 遇到问题先查 03-Knowledge，再查外部资源
- 优先使用 [[bb-browser]] 或 [[autocli]] 搜索社交平台

### 步骤 3: 分层沉淀
```
沉淀层级：
├── 主页.md ← 所有工作流精简简介
├── 工作流详情文档 ← 完整流程细则
└── 日志/归档 ← 执行记录
```

### 步骤 4: 记忆固化
- 定稿工作流存入 Hermes 长期记忆
- 后续任务优先匹配既定规则

## 🧠 三、模型编排策略

### 3.1 模型选择优先级

```
主模型链（按优先级降序）:
  1. **mimo**        ← 主入口，通用对话
  2. **deepseek**    ← 编码、复杂推理任务
  3. **glm**         ← 免费兜底，轻量任务
  4. **siliconflow** ← 免费备选池
      ├── Qwen3.5-4B
      ├── GLM-Z1-9B-0414
      └── GLM-4-9B-0414
```

### 3.2 任务-模型映射

| 任务类型 | 首选模型 | 备选 | 说明 |
|----------|----------|------|------|
| 日常对话/写作 | mimo | deepseek | 低延迟优先 |
| 编码/调试 | deepseek | Cursor CLI | 需要强推理 |
| 知识问答 | deepseek | glm | 准确度优先 |
| 轻量任务 | glm-4-flash | siliconflow | 免费模型撑 |
| 困难任务 | Cursor CLI | deepseek | 委托外部工具 |

### 3.3 子任务委派（Delegate）

```python
简单任务 → delegate_task(glm-4-flash)   # 免费、快速
中等任务 → delegate_task(deepseek)       # 平衡性价比
复杂任务 → delegate_task(mimo)           # 最强推理
```

- 子代理支持最大 3 路并行
- 嵌套深度限制为 1 层
- 子代理结果异步返回，不阻塞主流程

## 📖 五、知识管理体系

### 5.1 六段式目录

```
E:\工作需要\hermes-wiki/
├── 00-Inbox/     ← 临时笔记、灵感速记
├── 01-Daily/     ← 每日日志、复盘
├── 02-Project/   ← 项目跟踪
├── 03-Knowledge/ ← 常青知识（本页所在地）
│   ├── concepts/  概念
│   ├── entities/  实体
│   ├── comparisons/ 对比
│   ├── queries/   问答
│   └── summaries/ 总结
├── 04-Resource/  ← 模板、素材
├── 99-Assets/    ← 附件、图片
└── 98-Archive/   ← 归档
```

### 5.2 变更管理
- 每次操作追加 `log.md`
- 更新页面 bump `updated` 日期
- 新页面必须加入 `index.md`
- 使用 `[[wikilinks]]` 互链，每页至少 2 个出链

### 5.3 自动同步
- 每日/按需 → `push.sh` / `push.bat` → GitHub
- 公开内容推送：`01-Daily/`, `03-Knowledge/`, `04-Resource/`, `index.md`
- 隐私内容（`00-Inbox/`, `98-Archive/`）不推送
